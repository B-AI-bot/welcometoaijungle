What You'll Build

Not a chatbot. Not a glorified ChatGPT wrapper. An actual agent ‚Äî something that:

  • Runs 24/7 without you touching it
  • Remembers context across conversations

The difference between a chatbot and an agent is simple: a chatbot responds, an agent acts. By the end of this tutorial, you'll have an agent that acts.

Step 1: Choose Your Platform

You have three main options for running a personal AI agent in 2026:

Option A: Clawdbot/OpenClaw (Our Choice)

Open-source, self-hosted, extremely flexible. This is what we use. It connects to multiple AI providers (Anthropic, OpenAI, Google, etc.) and gives you full control.

  • Pros: Full control, model-agnostic, active community
  • Cons: Requires setup, you manage infrastructure

Option B: Claude Projects + API

If you just want something working quickly, Claude Projects with the API can work. Less flexibility, but faster to start.

Roll your own with LangChain, CrewAI, or similar frameworks. Maximum flexibility, maximum complexity.

  • Pros: Total control
  • Cons: Significant development time

Our recommendation: Start with Clawdbot. It hits the sweet spot between ease of use and flexibility. The rest of this guide assumes you're using it.

Step 2: Set Up Your Infrastructure

Local Development (Start Here)

Before going to a server, test everything locally:

# Install Clawdbot
npm install -g clawdbot

# Verify installation
clawdbot doctor

# Run the setup wizard
clawdbot init

The setup wizard will walk you through:

  1. Connecting your AI provider (we recommend Anthropic Claude)
  2. Setting up a messaging channel (Telegram is easiest)
  3. Creating your first skill

Production Setup (For 24/7 Operation)

Once you've tested locally, move to a VPS for always-on operation. We use a $10/month Ubuntu VPS on Hetzner, but any provider works.

# SSH to your VPS
ssh root@your-server-ip

# Security first (don't skip this!)
# Disable password auth
sudo sed -i 's/PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo systemctl reload ssh

# Set up firewall
sudo apt install ufw -y
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw enable

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Install Clawdbot
npm install -g clawdbot
clawdbot init

Keep It Running with PM2

# Install PM2
npm install -g pm2

# Start Clawdbot
pm2 start $(which clawdbot) --name clawdbot -- gateway

# Save for auto-restart
pm2 save
pm2 startup

Your agent now survives reboots and crashes. Magic.

Step 3: Configure Your AI Provider

This is where the intelligence comes from. We use Anthropic's Claude because:

  • Best at following complex instructions
  • Excellent at coding tasks
  • Strong reasoning capabilities
# Add your API key
clawdbot models auth anthropic

# Or use OpenRouter for multiple models
clawdbot models auth openrouter

Pro tip: OpenRouter lets you route different tasks to different models. Use Claude for complex reasoning, cheaper models for simple tasks. Your wallet will thank you.

Step 4: Define Your Agent's Personality

This is the fun part ‚Äî and the most underrated. Your agent's personality lives in a file called SOUL.md. Here's a minimal example:

# SOUL.md

You are [NAME], a personal AI assistant for [YOUR NAME].

## My Style
- Clear and concise ‚Äî respect their time
- Honest ‚Äî say what you think, no sugarcoating
- Proactive ‚Äî suggest improvements, don't just execute

## What I Know About My Human
- Timezone: [YOUR TIMEZONE]
- Work: [BRIEF DESCRIPTION]
- Preferences: [COMMUNICATION STYLE, PET PEEVES, ETC.]

## Boundaries
- Never share private information

The more context you give, the better your agent becomes. We have about 2,000 words in our SOUL.md, covering everything from communication preferences to decision-making frameworks.

Step 5: Connect a Messaging Channel

Your agent needs a way to talk to you. Telegram is the easiest to set up:

  1. Message @BotFather on Telegram
  2. Create a new bot with /newbot
  3. Copy your bot token
  4. Add it to your Clawdbot config
# In your config.yaml
channels:
  telegram:
    enabled: true
    token: "YOUR_BOT_TOKEN"
    allowFrom:
      - YOUR_TELEGRAM_USER_ID

Security note: The allowFrom field ensures only you can message your bot. Don't skip this.

Step 6: Add Your First Skills

Skills are what make an agent useful. Start with these essentials:

Web Search

# Get a Brave Search API key (free tier available)
clawdbot skills add web-search

File Management

Your agent can read/write files in its workspace. This is how it maintains memory and creates documents for you.

Memory System

# Enable semantic memory (game-changer)
clawdbot skills add memory

With memory enabled, your agent remembers past conversations and can search through them. This is what separates a real assistant from a stateless chatbot.

Step 7: Create Your First Automation

Let's build something useful: a daily briefing that runs every morning.

# In your config.yaml
cron:
  morning-brief:
    schedule: "0 7 * * *"  # 7 AM daily
    prompt: |
      Good morning! Prepare my daily briefing:
      1. Check my calendar for today
      2. Review my task list
      3. Summarize any important news in my field
      4. Weather for my location
      Keep it concise ‚Äî I read this in 2 minutes.

That's it. Your agent now wakes you up with a personalized briefing every morning.

Common Mistakes to Avoid

1. Skipping Security

An AI agent with access to your life is powerful ‚Äî and dangerous if compromised. At minimum:

  • Use SSH keys, not passwords
  • Firewall everything
  • Use Tailscale for secure remote access
  • Regularly run clawdbot security audit --deep

2. Overcomplicating Early

Start simple. Get basic messaging working before adding 15 integrations. You can always add more later.

3. No Memory Strategy

Without memory, your agent forgets everything between sessions. Enable the memory system from day one.

4. Treating It Like ChatGPT

This isn't a chat interface ‚Äî it's an agent. Give it context, let it run, review the results. Don't micromanage every step.

FAQ

How much coding do I need to know?

Almost none for basic setup. Terminal commands and YAML editing are enough. For advanced customizations, some programming helps but isn't required.

Is my data safe?

If you self-host (as we recommend), your data never leaves your server except when calling the AI API. Use API providers with strong privacy policies.

Can I use this for work?

Yes, but check your company's policies on AI tool usage. Many people use personal agents for work tasks, but keep sensitive company data out of external APIs.

What if something breaks?

Join the community Discord/forums. Clawdbot has an active community of 100k+ users who've solved most problems you'll encounter.

How is this different from using Claude directly?

Three key differences: (1) persistent memory across sessions, (2) ability to take actions automatically, (3) always-on availability. Claude web is a chat interface; this is an autonomous agent.
